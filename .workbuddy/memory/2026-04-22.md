## 部署方式
1. 主机环境: 直接运行脚本（非容器化）
2. Docker Compose: `cd deploy/docker && docker compose up -d`
3. Kubernetes: `helm install domino ./helm/domino`

## 修复: Docker 构建错误 (requirements.txt not found)

**问题**: compose.yaml 中构建上下文改为项目根目录 `context: ../..`，但：
1. `.dockerignore` 排除了 `rest/*`，导致 `COPY rest/requirements.txt` 找不到文件
2. `COPY . .` 期望文件在构建上下文根目录

**修复**:
- `imageBuild/rest/Dockerfile.prod`:
  - 使用 `COPY requirements.txt .`（项目根目录已有副本）
  - 使用 `COPY rest/ .`（保留 rest/ 源代码目录）
- `.dockerignore`: 改为细粒度排除
  - 原来: `rest/*` (排除整个目录)
  - 现在: 只排除 `rest/__pycache__/*`, `rest/*.pyc`, `rest/.pytest_cache/` 等构建产物
  - 保留 `rest/` 源代码目录

**注意**: 项目根目录的 `requirements.txt` 是从 `rest/requirements.txt` 复制过来的副本
