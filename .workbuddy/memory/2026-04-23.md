# 2026-04-23 Domino 前端问题修复

## 问题1：前端灰屏（已修复）
**现象**：资源 200 OK，但 JS 报错 `TypeError: Object prototype may only be an Object or null: import_meta_env_placeholder`
**根因**：`@import-meta-env/unplugin` 构建时未替换占位符
**修复**：移除 `@import-meta-env/unplugin`，改用 Vite DefinePlugin

## 问题2：API_URL 未生效（已修复）
**现象**：前端请求发到 `http://localhost:8000` 而非配置的服务器地址
**根因**：Vite DefinePlugin 在构建时将 API_URL 写死进 JS，compose.yaml 运行时环境变量对静态文件无效
**修复方案：entrypoint.sh 内联注入**
1. `index.html` 添加 `<!-- __DOMINO_ENV__ -->` 占位符
2. `entrypoint.sh` 读取 `$API_URL` 环境变量，用 sed 替换占位符为内联 `<script>`
3. `environment.config.ts` 优先读 `window.__DOMINO_ENV__`，回退到 `import.meta.env`
4. `Dockerfile.prod`: ENTRYPOINT 设为 `/usr/local/bin/domino-entrypoint.sh`

### 修改文件（v2）
| 文件 | 变更 |
|------|------|
| `frontend/index.html` | 添加 `<!-- __DOMINO_ENV__ -->` 占位符 |
| `frontend/entrypoint.sh` | 用 sed 内联注入 env 到 index.html |
| `frontend/src/config/environment.config.ts` | 读取 window.__DOMINO_ENV__ |
| `imageBuild/frontend/Dockerfile.prod` | ENTRYPOINT 调用 domino-entrypoint.sh |
