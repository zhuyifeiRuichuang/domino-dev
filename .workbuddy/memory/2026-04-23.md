# 2026-04-23 Domino 前端灰屏问题修复

## 问题描述
前端构建后显示灰屏，资源加载状态码都是 200，但界面内容未展示。

## 问题根因
Dockerfile.prod 中 WORKDIR 为 `/build`，然后所有文件复制到该目录：
- `/build/index.html`
- `/build/vite.config.ts`
- `/build/tsconfig.json`

Vite 构建时，`vite.config.ts` 中的相对路径（如 `example: ".env.production"`、`tsconfig.json`）解析出错，导致 `index.html` 没有被正确处理（script 标签未替换为构建产物路径）。

## 修复方案
修改 Dockerfile.prod，将 WORKDIR 改为 `/build/frontend`：
- 所有文件路径保持原样（在 frontend 子目录下）
- Vite 能正确找到 `vite.config.ts`、`tsconfig.json`、`.env.production`
- 构建产物在 `/build/frontend/build`

### 修改的文件
1. `imageBuild/frontend/Dockerfile.prod`
   - WORKDIR: `/build` → `/build/frontend`
   - 构建产物路径: `/build/build` → `/build/frontend/build`
   
2. `frontend/vite.config.ts`
   - 移除 `root: __dirname`（不需要，在正确目录下构建时 Vite 自动识别）
