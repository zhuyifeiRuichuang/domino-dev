# 2026-04-23 Domino 前端问题修复

## 问题1：前端灰屏（已修复）
**现象**：资源 200 OK，但 JS 报错 `TypeError: Object prototype may only be an Object or null: import_meta_env_placeholder`
**根因**：`@import-meta-env/unplugin` 构建时未替换占位符
**修复**：移除 `@import-meta-env/unplugin`，改用 Vite DefinePlugin

## 问题2：API_URL 未生效（当前修复）
**现象**：前端请求发到 `http://localhost:8000` 而非配置的服务器地址（如 `172.16.0.30:8000`）
**根因**：Vite DefinePlugin 在构建时将 API_URL 写死进 JS 代码，compose.yaml 的运行时环境变量对静态文件无效
**修复方案：运行时注入 + 构建时回退**
1. **entrypoint.sh**：容器启动时从环境变量生成 `/usr/share/nginx/html/env.js`
2. **index.html**：在主 JS 前加载 `<script src="/env.js">`
3. **environment.config.ts**：优先读取 `window.__DOMINO_ENV__`，回退到 `import.meta.env`
4. **Dockerfile.prod**：
   - entrypoint.sh 作为 nginx 启动钩子
   - Nginx 禁用 `index.html` 和 `env.js` 缓存
5. **.env.example**：更新 API_URL 说明，提示需用浏览器可访问的地址

### 修改的文件清单
| 文件 | 变更 |
|------|------|
| `frontend/index.html` | 添加 `/env.js` 引用，script 路径改为 `/assets/index.js` |
| `frontend/entrypoint.sh` | 重写为运行时环境注入脚本 |
| `frontend/src/config/environment.config.ts` | 双层读取：window.__DOMINO_ENV__ → import.meta.env |
| `imageBuild/frontend/Dockerfile.prod` | 启动脚本集成、Nginx 缓存策略 |
| `deploy/docker/.env.example` | API_URL 使用说明更新 |
